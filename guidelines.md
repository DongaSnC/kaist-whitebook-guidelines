KAIST 경영백서(2021~2026) 집필 지침
확정일: 2026-04-13 (ver 1.0) | 최종 갱신: 2026-05-08 (ver 2.4)
우선순위: 본 지침 > 노션 운영 지침 > 작업자 즉석 지시. 충돌 발생 시 AI는 임의 판단하지 않고 작업자에게 보고한다.

1. 백서의 성격
기관 운영 실적을 기록·보존하는 자료적 문서. 헌정형·홍보형 성격 배제.

2. 편집 기조
성과 수식 표현 금지: "탁월한", "획기적인", "눈부신", "역대 최대", "선도적". 정성 평가는 외부 출처(평가보고서·언론·랭킹) 인용으로 대체. 모든 정책·프레임은 "2021~2026년 임기 중"으로 시기 한정.

3. 문체·표기
평어체(~이다/~한다). 기관 명칭은 "KAIST"(1인칭 금지). 외래어·약어는 첫 등장 시 풀어쓰기. 수치는 출처·기준일 병기, 불명확 시 [확인 필요]. 능동태 우선.

4. 글의 형식
문장: 서술형 산문 기본. 개조식 본문 금지. 단문과 접속문을 적절히 배합하며, 단락 전환 시 "이러한 운영을 바탕으로", "같은 맥락에서" 등 연결 표현을 명시한다.
단락: 하나의 주제, 3~6문장. 2문장 이하는 앞뒤와 합친다. 첫 문장에 핵심 정보.
소제목: 명사형으로 끝맺음. 서술형 금지. 본문 최소 3문장 이상.
소절-중간절 구조: 소제목들이 선후 관계면 연결 표현으로 흐름을 이어가고(유형 A), 병렬 나열이면 중간절 도입부에서 상위 맥락을 먼저 제시한다(유형 B).
분량: 소절(x.x.x) A4 0.3~1쪽 / 중간절(x.x) A4 1~3쪽.
표·수치: 정량 데이터 3개 이상이면 표 우선. 표에 기준 시점·출처 필수. 단위는 첫 등장 시 명시.
제2부 절 구조: ① 추진 배경 → ② 주요 추진 과제 → ③ 핵심 성과 → ④ 대표 사례.

5. QAIST·인물·민감 항목
QAIST: 1부 4장에서 본격 소개(5~6쪽). 2부 각 장 도입부에서 연관성 1~2단락만 언급. 1대1 대응 구조 금지. 세부 슬로건은 실적 확인 시에만 인용.
인물: 총장 사진은 발간사 1장만. 어록·인터뷰 인용 금지. "총장의 비전/철학/리더십" 표현 금지. 보직자 명단은 부록에 직위·재임기간만.
민감 항목: 안전사고는 4부에 사실 위주 2~3단락(발생→경위→대응→제도개선), 책임 평가 금지. QS 사안은 사실관계 범위 내 기록. 차기 총장 선임·재선 시도는 일체 언급하지 않는다.

6. 출처·주석
각주 방침 미확정 — 중간절 말미에 ※각주 방침 확정 후 출처 형식 통일 요망 1회만 표기. 인라인 출처는 모든 문단 뒤에 (출처: 보고서명 제X장 X절 X항) 형식으로 표기(절 말미 일괄 금지). 수치 불일치 시 기관운영평가 보고서를 주 출처로 사용.
출처 위치 특정 원칙: 출처 표기 시 반드시 장·절·항 제목을 기준으로 위치를 특정한다. AI는 grep 검색 후 해당 줄 앞에 등장하는 가장 가까운 장·절·항 제목을 확인하여 출처에 기입한다. 줄 번호만으로 위치를 추정하거나 장·절 제목 확인 없이 출처를 기입하지 않는다. 장·절·항 제목이 확인되지 않으면 (출처: 보고서명 — 위치 확인 필요)로 표기하고 담당자에게 확인을 요청한다.
페이지 번호: 장·절·항 제목으로 위치가 특정된 경우에 한해 추가 표기. 확인되지 않으면 생략. 페이지 번호 최종 확인은 담당자가 원본 PDF에서 직접 수행한다.

7. 이미지 플레이스홀더
이미지로만 존재하는 모든 시각 자료는 텍스트 재현 없이 아래 형식으로 처리.

사진·그래프: [사진] 캡션명 (출처: 보고서명 제X장 절명, p.XX) + ※ 담당자: 원본 보고서에서 이미지 삽입 요망
조직도·다이어그램: [그림] 캡션명 (출처: 보고서명 제X장 절명, p.XX) + ※ 담당자: 원본 보고서에서 이미지 삽입 요망
소제목 다음 내용 전체가 이미지: [그림] ▶ 소제목명 (출처: 보고서명 제X장 절명, p.XX, 원본 확인 필요) + ※ 담당자: 원본 PDF 해당 페이지 캡처 후 삽입 확인 요망

구글 드라이브 이미지 호출은 수행하지 않는다.

8. 세션 운용
세션 단위는 중간절(x.x) 하나. 세션 시작 시 AI 수행 순서: ① web_fetch로 지침 파일 호출(https://raw.githubusercontent.com/DongaSnC/kaist-whitebook-guidelines/main/guidelines.md) → ② 현황 리포트 → ③ 이번 세션 작업 범위 보고(제목 기준) → ④ 제안사항 제시 → 작업자 승인 후 집필 시작. 원고 완성 후 교정 리포트(요약표만) 먼저 제출, 작업자 검토 후 승인. 세션 종료 시 인계 프롬프트 작성.
작업 대상 식별 원칙: 절은 반드시 제목 기준으로 식별한다. 절 번호는 목차 변경 시 연쇄적으로 바뀌는 불안정한 식별자이므로 기준으로 쓰지 않는다. 번호는 현황 추적의 보조 정보로만 사용한다. 인계 프롬프트에서 다음 작업 대상은 "제목(현재 절 번호)" 형식으로 표기한다. 예: "의과학대학원 문지캠퍼스 이전(현재 5.4)".
노션 연동: 노션 MCP 연동은 프로젝트 관리자(김택원)만 수행한다. 팀원은 세션 시작 시 위 web_fetch 방식으로 지침을 읽는다. 지침 수정 사항은 슬랙 운영 공지 스레드에 올리면 관리자가 GitHub에 반영한다.
모든 모순·결정사항은 AI가 임의 판단하지 않고 작업자에게 보고한다. 원자료 미수록 항목은 스킵 대신 📌 편집자 메모: 원자료 미수록. 총장실 자료 추가 요청 필요. 메모 후 보류.
지침 변경 사항은 작업자 승인 → 슬랙에 작업현황 공지 → 분리된 슬랙 포스트로 지침 업데이트 사항 공지 순으로 처리.

9. 세부 집필 결정사항 (노션 5-1항 참조)

독서왕: CYYP와 분리하여 도서관 사업으로 서술 (5.1.4)
실패포상(실패왕): 5.1.3 도전·실패 장려 문화 절에 서술
각주 방침 메모: 중간절 말미에 1회만 표기
원자료 미수록 항목: 스킵 대신 편집자 메모 후 보류
A그룹 소절 처리: 내용이 지나치게 적거나 지엽적인 소절은 상위 중간절 본문에 흡수 서술. 목차 작업 메모에 "▶ 본문 통합"으로 표시.
